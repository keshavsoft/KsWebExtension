# Content Scripts Injection Model

Web extensions operate in an isolated execution sandbox. They have access to the DOM, but their global JavaScript environment (`window`) is separate from the webpage's own JavaScript environment. To make libraries like KSTable available to the pages, we inject them directly into the page context using script tags.

document_start Early Initialization

Executes as soon as the document is created, before any stylesheets or DOM content are loaded or parsed.

Main Purpose: Declares and sets up core web components quickly. This avoids page flickering when custom HTML tags are encountered.

FILES INJECTED:
src/KSComponents.js
document_end Late Injection

Executes after the HTML document is fully parsed and loaded, but before subresources like images or iframe contents finish loading.

Main Purpose: Loads layout scripts, headers, tables rendering structures, and vertical list renderers onto the page.

FILES INJECTED:
src/ksheader.js
src/kstablecomp.js
src/kstableonly.js
src/ksvertical.js

# Web Accessible Resources

For security, the browser requires any files that will be injected into a page context to be explicitly declared inside `manifest.json` under web_accessible_resources.

If you add any new file that needs to be loaded directly on the page, make sure to add it under the `resources` list in the manifest file, otherwise the browser will block the load with a Security Violation.
