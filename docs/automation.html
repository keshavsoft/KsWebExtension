Automation & Release Management

How the build scripts pull code from sibling workspace repos, create archive backups, and generate the Firefox extension package.

# Automation Script: update-src.js

The `update-src.js` script handles syncing workspace resources automatically. It contains mapping definitions for all the compiled libraries, searching in their respectives `/dist` or `/public` folders to find the latest version directories.

update-src.js (Source Code Configuration) 
// Sibling directories mapping to the files in src/
const fileMappings = [
  {
    repo: 'tailwind-header-dom',
    subDir: 'public',
    prefix: 'v',
    files: ['ksheader.js', 'ksheader.min.js']
  },
  {
    repo: 'ks-web-comp-table',
    subDir: 'dist',
    prefix: 'v',
    files: ['KSComponents.js']
  },
  {
    repo: 'tailwind-table-dom-comp',
    subDir: 'dist',
    prefix: 'v',
    files: ['kstablecomp.js']
  },
  {
    repo: 'tailwind-table-dom-comp-show',
    subDir: 'dist',
    prefix: 'v',
    files: ['kstableonly.js']
  },
  {
    repo: 'tailwind-vertical-dom',
    subDir: 'dist',
    prefix: 'v',
    files: ['ksvertical.js']
  }
];

# Release Cycle Steps

Follow this exact checklist when preparing and uploading a new release:

  1. 1

    Update manifest.json version

    Open manifest.json and change the "version" field (e.g. "2.28"). Do not touch the package.json version; the script will handle syncing it.

  2. 2

    Run update command

    Run npm run update-src. This copies your files to the archives folder, updates the extension source files, and builds the new zip package in the root directory.

  3. 3

    Update release notes

    Open upload.md and draft the version release notes. Copy older notes from upload.md into archive/upload_history.md to keep history tidy.

  4. 4

    Upload zip package

    Log in to the Firefox Add-on Developer Hub, upload the new .zip file, and paste the release notes from upload.md.
